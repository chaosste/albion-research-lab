# oh-my-codex integration

This repo vendors [oh-my-codex](https://github.com/chaosste/oh-my-codex) as a git submodule at `./oh-my-codex`.

The submodule is pinned to the same commit as the local checkout at `/Users/stephenbeale/Projects/oh-my-codex`.

Project-local OMX surfaces live at:

- `./.codex/` (agents, prompts, skills — tracked in git)
- `./.omx/` (runtime state — gitignored, created by setup)
- `./AGENTS.md` (generated by setup — gitignored)

## First-time setup

```bash
git submodule update --init --recursive
cd oh-my-codex
npm install
npm run build
cd ..
uv sync
./omx setup --scope project --force --verbose
```

## Recommended local launcher

Use the repo-local wrapper from the repo root:

```bash
./omx doctor
./omx setup --scope project --force --verbose
./omx explore --help
```

The wrapper resolves to:

- `node ./oh-my-codex/dist/cli/omx.js`

## Updating the submodule

When you update `/Users/stephenbeale/Projects/oh-my-codex` and want to pin a new version here:

```bash
cd oh-my-codex
git fetch origin
git checkout <commit-or-tag>
npm install
npm run build
cd ..
git add oh-my-codex
git commit -m "Pin oh-my-codex submodule to <commit-or-tag>"
```

Then rerun project setup if needed:

```bash
./omx setup --scope project --force --verbose
```
